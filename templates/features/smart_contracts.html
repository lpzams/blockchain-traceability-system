智能合约说明
什么是供应链智能合约？

智能合约是在区块链上自动执行的协议，可以自动验证交易条件并执行相应操作。

合约特点
  • 自动执行：无需第三方，条件满足自动执行
  • 透明可信：所有交易条款公开，不可篡改
  • 所有权验证：系统自动验证产品所有权
  • 多方协议：支持供应链各环节的合作协议
合约生命周期
  1. 创建：定义交易条款和参与方
  2. 待签：等待接收方确认
  3. 生效：双方同意，合约正式生效
  4. 执行：系统验证条件并执行
  5. 完成：交易完成，合约终止
风险控制
  • 自动验证交易双方的有效身份
  • 检查产品所有权的真实性
  • 记录所有交易步骤于区块链
  • 不可篡改的交易历史
技术架构
合约验证流程
1. 输入验证
   ├─ 检查产品是否存在
   ├─ 验证当前所有者身份
   └─ 验证接收方身份

2. 所有权检查
   ├─ 查询产品当前所有者
   ├─ 与合约中的发送方匹配
   └─ 返回所有权验证结果

3. 合约生成
   ├─ 生成唯一合约ID (SHA256)
   ├─ 计算合约校验和 (SHA256)
   └─ 记录合约到区块链

4. 自动执行
   ├─ 所有权验证成功
   ├─ 自动转移产品所有权
   └─ 记录交易事件
数据结构
合约对象 {
    contract_id: string,           // 合约ID (SHA256)
    from_user_id: string,          // 发送方ID
    to_user_id: string,            // 接收方ID
    product_id: string,            // 产品ID
    terms: {                        // 交易条款
        price: number,             // 交易价格
        delivery_days: number,     // 交付期限
        notes: string             // 备注信息
    },
    status: string,                // 合约状态 (pending/accepted/rejected/completed)
    created_at: string,            // 创建时间
    expires_at: string,            // 过期时间 (30天有效期)
    checksum: string              // 合约校验和 (SHA256)
}
